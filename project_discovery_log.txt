- Project Discovery Log - Retail Customer Segmentation
- Purpose: Central source of truth for key insights, technical decisions, and business findings across the project lifecycle.

- Data Ingestion
  - Loaded the Online Retail II dataset from the local `./data/` path into a Pandas DataFrame.
  - Performed initial cleaning to improve analytical reliability:
    - Removed rows with missing customer identifiers.
    - Removed canceled orders where Invoice begins with `C`.
  - Persisted cleaned transactional data into a local SQLite database `retail.db` in table `transactions` for reproducible downstream analysis.

- Feature Engineering (RFM Logic)
  - Built customer-level RFM features by grouping transactions on `CustomerID`.
  - Defined snapshot date as one day after the latest `InvoiceDate` in the cleaned dataset.
  - Calculated:
    - Recency: Days between snapshot date and each customer’s most recent invoice date.
    - Frequency: Number of unique invoices per customer.
    - Monetary: Total spend per customer calculated as sum of (`Quantity * Price`).
  - Applied a quality filter to keep only customers with positive Monetary values (`Monetary > 0`).

- Preprocessing
  - Assessed skewness in Recency, Frequency, and Monetary using `seaborn.histplot`.
  - Applied `np.log1p` log transformation to reduce right-skew and dampen outlier impact.
  - Applied `StandardScaler` to normalize feature scales so K-Means distance calculations are balanced across all RFM dimensions.

- Clustering
  - Executed Elbow Method (K=1 to K=10) using K-Means inertia values.
  - Observed an elbow bend around K=2.
  - Strategic decision: selected K=4 to create more actionable and business-usable customer personas, rather than a minimal-cluster technical optimum.
  - Added cluster labels back to the original (non-log) RFM DataFrame for interpretability.
